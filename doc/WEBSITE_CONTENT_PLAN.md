# 个人网站内容规划

> 网站定位：个人技术主页，用于展示个人背景、持续积累的技术笔记、公开项目与联系方式。  
> 第一版原则：内容真实、结构简洁、持续更新；暂不加入登录、评论、管理后台等动态功能。

## 1. 信息架构

网站采用五个一级入口，其中首页负责概览，四个核心板块承载完整内容：

```text
首页 Home
├─ 关于我 About
├─ 笔记 Notes
├─ 项目 Projects
└─ 联系我 Contact
```

| 入口 | 要回答的问题 | 主要内容 |
| --- | --- | --- |
| 首页 | 我是谁、网站有什么 | 身份、关注方向、最新笔记、精选项目、快捷链接 |
| 关于我 | 我的经历和能力 | 简介、教育经历、关注方向、技能、当前状态 |
| 笔记 | 我在学习和记录什么 | Markdown 文章、分类、标签、文章详情 |
| 项目 | 我做过什么 | GitHub 公开项目卡片、项目介绍、仓库链接 |
| 联系我 | 如何联系我 | 邮箱、GitHub、后续可加入简历和学术主页 |

## 2. 首页

首页不重复完整简历，而是提供简洁的个人定位与内容入口。

### 2.1 首屏内容

```text

通过个人主页记录技术学习、项目实践与成长过程。
```

首屏操作按钮：

- 阅读笔记 → `/notes`
- 浏览项目 → `/projects`
- GitHub → `https://github.com/Orange0618`

### 2.2 首页内容区块

1. **关注方向**：使用 3–4 个标签展示软件开发、AI 辅助编程、物联网、数码科技。
2. **最新笔记**：展示最近更新的 3 篇 Markdown 笔记，包含标题、摘要、分类、日期与链接。
3. **精选项目**：优先显示 3–6 个有 README、截图或明确功能说明的项目；提供“查看全部项目”入口。
4. **当前状态**：用简短列表展示正在做的事，例如整理嵌入式笔记、探索 AI 编程工具、学习 STM32 与 Linux 应用开发。

## 4. 笔记

### 4.1 内容来源与渲染

笔记由仓库中 `resource/note/` 目录下的 Markdown 文件提供。网站构建时读取文件、解析 Frontmatter，并生成文章列表与详情页。

当前已有内容集中在嵌入式方向，例如：

```text
resource/note/嵌入式/
├─ 电路分析.md
├─ 电子基础.md
├─ STM32（标准库）.md
├─ STM32（HAL库）.md
├─ PCB.md
├─ Linux.md
└─ Linux智能车库项目.md
```

后续可按实际内容增加 `ai-coding`、`software-development`、`iot` 等分类；不要预先创建没有文章的空分类。

### 4.2 笔记列表页

列表页应支持：

- 分类浏览；
- 标签筛选；
- 按发布时间或更新时间排序；
- 标题、摘要、分类、标签、更新时间的文章卡片；
- 文章数量增加后再加入全文搜索。

### 4.3 文章详情页

每篇文章至少提供：

- 标题、摘要、发布时间和最后更新时间；
- 自动生成的文章目录；
- 代码语法高亮、表格、图片与 Mermaid 图；
- 上一篇/下一篇导航；
- 返回笔记列表按钮；
- 可选的“在 GitHub 上编辑”链接。

建议统一使用 Frontmatter：

```markdown
---
title: STM32 HAL 库学习笔记
date: 2026-07-15
category: 嵌入式
tags:
  - STM32
  - HAL
description: 记录 STM32 HAL 库的常用模块和学习要点。
---

# STM32 HAL 库学习笔记
```

## 5. 项目

### 5.1 数据来源与展示规则

项目数据来自 GitHub 主页：<https://github.com/Orange0618>。

实现时读取公开、非 Fork、非归档的仓库，并为每个仓库生成独立项目卡片。每张卡片至少包含：

```text
项目名称
一句话简介（优先使用 README，其次使用仓库描述）
主要语言或技术标签
最后更新时间
查看源码链接
```

卡片的“查看源码”应直接链接到对应 GitHub 仓库。若仓库缺少描述或 README，应显示简洁的占位说明，并在后续完善仓库资料后自动改善展示。

### 5.2 项目页层级

```text
项目页
├─ 精选项目：有完整说明或代表性的仓库
├─ 全部公开项目：每个符合规则的仓库一张卡片
└─ GitHub 主页入口
```

首页只放精选项目，项目页展示全部符合规则的公开项目，避免首页信息过于拥挤。

## 7. 页脚与后续扩展

页脚建议包含：

```text
© 2026 王茂林 · Built with Vue / VitePress · GitHub · Last updated
```

第一版不实现以下功能：

- 登录注册、评论与在线聊天；
- 在线上传 Markdown、数据库与管理后台；
- 自动播放音乐、大量 3D 动画、无意义的图标墙；
- 空白的论文、奖项或成果页面。

当出现足够的论文、专利、竞赛或研究经历时，再在“关于我”之外新增“成果 / Publications”板块；当需要跨设备在线写作和发布笔记时，再考虑引入 Serverless API、对象存储和身份认证。
